CSS Backgrounds

CSS supports several useful background related properties.

  • background-color:
    assign the background color
  • background-image: url()
    applies an image as the background
  • background-repeat: repeat | repeat-x | repeat-y | no-repeat
    determines how a background iamge will repeat display
  • background-position: <percentage> | <length> | left | center | right
    the position can be assigned with either pixels or percent
    background-position: 100px 100px;
    using pixels, you can position the x y starting point for the background image. 0px 0px is the default
    background-position: 50% 0px;
    using percentages you can center a background image, or with varying percentages can create a parallax background effect

The pattern to follow when using the background property to define several space delimited values at once:
background: <background-color> | <background-image> | <background-repeat> | <background-attachment> | <background-position>

Sprites

Each image on your website requires a separate request/response between the client and server. Using one larger image instead of several smaller images can result in a more effiecient client/server transaction, since it reduces the number of requests/responses that need to occur.

This is especially useful for images that are NOT an important part of your foreground content, but sprites perfect for icons, textures and images used purely for aesthetics

  1. use a sprite generator or an image program like photoshop to create a strip or grid of images. this is also an ideal time to determine the correct offsets
  2. use an HTML tag ('div' or 'p' or etc) any block element
  3. define a fixed height and width (large enough to show the image you need to display)
  4. use the url and position offset values for example:
    • background: url("myimage.jpg") xOffset yOffset;
    • background: url("myimage.jpg") 0px 0px;
      places myimage.jpg in the top left corner ( 0px 0px )
    • background: url("myimage.jpg") 50px 0px;
      places myimage.jpg at the top, but shifts the start of the image 50 pixels to the right (offset 50px)
    • background: url("myimage.jpg") -50px 0px;
      places myimage.jpg at the top, but shifts the start of the image 50 pixels to the left (offset -50px)

Advantages of using sprites

  • fewer http requests between server and client
  • a single composite image is almost always smaller in filesize compared to the sum size of the images made separately
  • perfect for icons, buttons, and graphics that contribute to the aesthetic of the page

Disadvantages of using sprites

  • not all images should be displayed as backgrounds! important content graphics should always use the img tag. this way the images will contribute to search engine optimization and maximize accessability
  • time spent building the composite sprite image(s) increases
  • time coding and maintaining the CSS increases

CSS Reset

An excellent way to instruct a browser to set attributes to predictable values

Custom created and self-hosted, or public domain CDN hosted options and everything in between!

  • Content is now predictable in most every browser
  • Minimizes odd behaviour in rendering
  • Easy to create custom rules for CMS-controlled sites

There are some excellent resets out there, some of the better ones can be found at cssreset.com

  • Doesn't just reset, but takes advantage of some of the built-in features of various tags

  • Can work embedded as well

CSS Positioning

Building proper CSS-based layouts requires an understanding of how positioning works, which is based largely on the Box Model concept.

A lot has been written about the fabled Box Model and it's lack of support in certain browsers.

It actually comes down to an interpretation of the box model concept that was implemented differently on different browsers.

Box model basics:

  • All elements are built around the concept of a box
  • Every element is contained in a Box of some sort or another
  • Each element has a content core, surrounded by optional padding, borders, margins
  • These are the 'building blocks' of a CSS layout, with both inline and block-level elements having these properties
  • Width refers to left inner edge to the right inner edge
  • Height refers to the top inner edge to the bottom inner edge
  • This can cause problems if elements are near one another with considerable padding or margins
  • Paddings and borders add to the dimensions of the box model.
  • Additionally, margins will collapse vertically, but not horizontally. This can lead to some confusion when trying to build out a well-positioned page

In the following example:

<body>
	<div>
		<p>This is some text</p>
	</div>
</body>
  • The <p> block is contained within the <div> block, which is, in turn, contained inside the <body> block.
  • This is crucial for visual layout, as this will determine how your layout looks.
  • Inline blocks are not as crucial, as they will flow alongside other elements and have little outcome on layout.

In order to have some control over how these boxes are placed, block boxes (like div or p or h1) act as containing blocks to the content and blocks inside.

Containing blocks are used to determine the position and dimensions of the elements within it.

<div id="main">
  <p style="width:25%;">
    This paragraph will be 25% the width of the parent div,
    not 25% of the page
  </p>
</div>

Positioning Details

  • Normal Flow - these are generally text that 'flows' from left to right (in Western society). Normal flow is the default scheme with block elements flowing from the top downwards. Inline elements flow left to right
  • Normal Flow also sees your vertical margins collapsing. Only the larger of the two margins are used when calculating distance between the two.
  • Non-replaced element - these elements are actually contained inside the document.
    • Example: a paragraph of text.
  • Replaced element - these are not actually contained in the document, but are linked to.
    • Example: <img> or <input>

Floating and Positioning

  • In order to have your content show in an appropriate location, you will need to either float or position your containers.
  • Positioning is fairly strightforward and has been around for a long time.
  • Declare exactly where an element box will appear compared to where it would normally be
  • 4 kinds:
    • Static: Generated as normal
    • Relative: The box is offset by some distance
    • Absolute: The box is no longer part of the flow of the document and is placed with respect to its containing block.
    • Fixed: Similar to absolute, but it is attached to a specific location
  • Floating was introduced in CSS1 and gives rise to a lot of difficulties for many users.
  • Similar to <img src="img.gif" align="right"> which 'floats' the image to the right of the content.
  • Other content will float around floated elements.
  • Consecutive floats (with similar alignment like right or left) will attempt to float using the same top-line, but will collapse to another line if space does not permit
  • Consecutive right floats will appear in reverse order on the final render
  • Other floated areas can create some interesting effects if DIVs are not floated correctly
  • Backgrounds can cause havoc with floated elements, as much of the time, you will see the background coming through
  • To make sure elements don't fall into a floated element, you can use clear: left | right | both;

Layout Basics

  • The root element (html) is the container for the viewport, or the window which you see the document. Variable size!
  • In order to lay out content on the screen, there needs to be a container that you can place your content into.
  • For a single column, vertical stack, this is fairly easy: just place all the content in the container.
  • For more complex layouts, (more than 1 column), you will need to create a container to hold your different content areas.
  • This is where a DIV comes in
  • A DIV is an area that can be defined in CSS as having the properties of a box
  • That box will act as a container for whatever goes inside
  • DIV elements can be floated or fixed. Similarly, elements withing those divs can also be positioned relatively or absolutely

A proposed layout building methodology

  1. design without limits (dont design with 'limitations' of CSS/HTML in mind)
  2. set your test browser to default zoom size
  3. create divs for each major region of content
  4. if more than one element is part of a group, consider diving it up
  5. apply box model dimension properties
  6. assign absolute and fixed positioning where applicable
  7. assign float, relative positioning where applicable
  8. add / remove quantities of content to see how layout behaves
  9. change height and width of browser to see how layout behaves
  10. if all else fails, consider using one of several templates made available online. if you use someone else's code, be sure to give credit where credit is due by adding a comment explaining where you found the original code.

Summary

  • If your page includes several icons or generic images that aren't a part of your foreground content, consider using sprites
  • Apply a CSS Reset of your choice to bring the various browser defaults into alignment, making for a more predictable and consistent page layout
  • Padding and Borders add to the dimensions of the CSS box model
  • Horizontal meeting margins are added, vertical meeting margins are collapsed
  • Practice all the CSS positioning techniques: float, absolute, relative, fixed
  • Use divs or HTML 5 semantic tags to define page layouts. If you have difficulty building your desired layout, consider searching for CSS layout examples online
  • Use unordered lists ul for site navigation. This is the best semantic option, ideal for screen readers and search engines.
